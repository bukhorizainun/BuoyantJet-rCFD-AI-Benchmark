# Dataset Description

This document defines the canonical layout consumed by every script and
notebook in this repository.

## 1. Top-level layout

```
data/
├── raw/                # Direct outputs from Fluent
├── processed/          # Cleaned NumPy / CSV arrays
├── cfd_snapshots/      # 630 CFD snapshot images
├── rcfd_snapshots/     # 600 rCFD snapshot images
├── metadata/           # Per-run YAML/JSON manifests
└── README.md
```

## 2. `data/raw/`

Files written directly by ANSYS Fluent. Examples:

| File                                | Origin            | Notes                              |
|-------------------------------------|-------------------|------------------------------------|
| `CFD_temperature.out`               | CFD reference run | mean-T monitor, ASCII columns      |
| `CFD_start_energy.out`              | CFD spin-up       | energy trace                       |
| `rCFD_temperature.out`              | rCFD replay       | mean-T monitor                     |
| `rCFD_balance.out`                  | rCFD replay       | energy balance                     |
| `fluent_*.trn`                      | Fluent            | console transcripts                |
| `rcfd_db.meta.toml`                 | rCFD prep         | DB metadata                        |

`*.out`, `*.trn`, `*.cas.h5`, `*.dat.h5` are all git-ignored.

## 3. `data/cfd_snapshots/`

PNG/JPG frames exported by Fluent during the CFD reference run.

| Property         | Value                              |
|------------------|------------------------------------|
| Naming           | `CFD_T_path_XXXX.jpg`              |
| Index range      | 0000 .. 0629                       |
| Count            | 630                                |
| Time-step        | 0.2 s per frame                    |
| Content          | temperature field on a fixed plane |

## 4. `data/rcfd_snapshots/`

| Property         | Value                              |
|------------------|------------------------------------|
| Naming           | `rCFD_BJet_XXXX.jpg`               |
| Index range      | 0000 .. 0599                       |
| Count            | 600                                |
| Time-step        | 0.2 s per frame                    |
| Content          | replayed temperature field         |

CFD has **30 extra frames** at the start that correspond to the spin-up tail
not replayed by rCFD; the comparison scripts handle this offset explicitly.

## 5. `data/processed/`

NumPy / CSV products generated by `scripts/prepare_ai_dataset.py` and
`scripts/export_numpy_arrays.py`:

| File                       | Shape / format                     |
|----------------------------|------------------------------------|
| `cfd_T_field.npy`          | `(N_cfd, H, W)` float32            |
| `rcfd_T_field.npy`         | `(N_rcfd, H, W)` float32           |
| `cfd_T_mean.csv`           | `t [s], T_mean [K]`                |
| `rcfd_T_mean.csv`          | `t [s], T_mean [K]`                |
| `cfd_vs_rcfd_errors.csv`   | per-step MAE / rel-err             |
| `dataset_manifest.yaml`    | hash + provenance                  |

## 6. `data/metadata/`

YAML/JSON manifests describing each run (case id, mesh, BCs, solver settings,
checksum). Manifests **are** tracked in git so the dataset provenance is
preserved even when bulk files are not.

## 7. `sample_data/`

Tiny demo subset (a few CFD/rCFD frames + truncated `.out`) used by the
notebooks for quick smoke tests, fully tracked in git.

## 8. Provenance & honesty note

All bulk data referenced here was produced by the local CFD/rCFD runs
described in `docs/cfd_setup.md` and `docs/rcfd_workflow.md`. No synthetic or
augmented data is present. If/when AI training data is generated, it will
live under `data/processed/` and be clearly tagged in `dataset_manifest.yaml`.
